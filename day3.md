## Задание на день 3. Применение python для работы с elasticsearch

### Отчёт по пз
Демонстрация работоспособности набора скриптов, упакованных в docker. Выполнить все задания, используя python.

### Ход работы
1. День 1 закончился на моменте встраивания [fastapi-todo-lr1](https://gitwork.ru/anetto/fastapi-todo-lr1) в композ. Обратите внимание, что мы работаем с новым docker compose (который часть докера), а не с устаревшим docker-compose. Теперь начинаем это приложение дорабатывать. Документация [fastapi](https://fastapi.tiangolo.com/), вверху есть переключатель языка.

1. Работу вести в автоматически пересобираемым compose. Разобраться, как сделать так, чтобы после изменения кода он автоматически пересобирался и не требовал от вас действий - вы в pycharm изменили, в браузере уже новая версия (возможно, после F5). Данные хранить в подкаталоге data своего проекта. Каталог data добавить в .gitignore.

1. Реализовать подсоединение к эластику и сохранение текста задачи в эластике. То есть текст задачи сохраняется и в sqlite (или postgresql), как сейчас, так и индексируется в эластике. Код для работы с эластиком оформлять в виде функций, вызываемых внутри http-ручек. То есть не писать код сразу в функции home.

1. Применять фильтр грифов секретности (известные вам грифы превращает в надписи "не интересно", "не интерессно", "не интересссно"). Токенизатор должен быть настроен для русского языка, фильтр токенов - на стоп-слова русского языка и три любых разных имени из вашей группы.

1. Реализовать поле ввода для поиска с кнопкой "поиск". В поле ввода задаётся строка, в результате поиска на странице должны показаться только тудушки, в которых эта строка есть. Искать нестрогое соответствие (то есть по запросу "красный" должно найти и "красный", и "красная", и другие вариации).

1. Добавить отдельный скрипт генерации 20 тудушек со случайными заголовками, который должен вызывать ручку создания новой тудушки. Сделать по аналогии с dockerfile_generate, но обязательно писать на python (не bash, использовать библиотеку requests или любую другую). Добавить в ридми порядок запуска этого скрипта через Docker.

1. Сохранять дату создания заметки в эластике. Реализовать поиск всех заметок, созданных после указанной даты.

1. Реализовать корректное изменение и удаление документа из эластика при изменении и удалении тудушки

1. В режиме редактирования тудушки есть тег "планы" / "учёба" / "личное". Реализовать его сохранение в эластике и кнопку поиска по тегу (вернуть все тудушки заданного тега - можно через 3 разных кнопки).

1. В режиме редактирования тудушки сделать кнопку загрузки файла, который индексировать в эластике.

1. Реализовать, чтобы кнопка поиска искала в том числе по содержимому загруженных файлов.
